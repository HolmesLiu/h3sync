# 瑞普数据同步系统 - 实施计划（一期）

## 1. 项目里程碑
### 里程碑 M1：基础框架与数据库（第 1 周）
目标：系统可启动、可登录、数据库可初始化
1. 搭建 Go 单体工程结构
2. 接入 PostgreSQL 与迁移工具
3. 完成后台登录与会话管理
4. 落地核心元数据表与日志表

交付物：
1. 可运行服务（dev 环境）
2. 初版 DDL 与迁移脚本
3. 登录页 + 管理后台骨架

### 里程碑 M2：氚云同步闭环（第 2 周）
目标：完成表单配置与同步执行
1. 实现 H3Connector（认证、分页拉取）
2. 实现动态建表与 upsert
3. 实现自动/手动同步
4. 实现增量游标机制
5. 写入同步日志

交付物：
1. 可同步至少 3 个试点表单
2. 后台可查看同步状态与错误

### 里程碑 M3：查询 API 与后台管理完善（第 3 周）
目标：对外查询可用，后台可配置 API Key 与导出
1. 实现 API Key 管理（名称、备注、权限、过期）
2. 实现查询 API（过滤、分页、基础关联）
3. 响应中返回表单/字段备注（AI 友好）
4. 实现数据筛选与导出
5. 写入 API 查询日志与后台审计日志

交付物：
1. 可供 AI 调用的稳定查询 API
2. 完整后台管理功能

### 里程碑 M4：联调与上线（第 4 周）
目标：试运行稳定，具备上线条件
1. 压测与稳定性验证
2. 数据准确性抽样对账
3. 错误场景与重试验证
4. 编写运维手册（部署、备份、恢复、监控）
5. Ubuntu 上线部署

交付物：
1. 上线版本
2. 运维手册与交付文档

## 2. 任务分解（WBS）
1. 需求确认
- 固化字段映射与备注管理规范
- 固化 API 查询协议

2. 研发
- 核心模块开发（admin/sync/connector/query-api/audit）
- 单元测试与集成测试

3. 联调
- 氚云接口联调
- 数据抽样核验

4. 上线
- 生产部署
- 运行观察与参数调优

## 3. 测试计划
1. 功能测试
- 登录、配置、同步、查询、导出

2. 同步准确性测试
- 全量后增量
- 重复执行幂等
- 异常中断后续跑

3. 权限安全测试
- API Key 过期
- 未授权表单访问拦截
- 后台审计记录完整性

4. 性能测试
- 50 表单周期同步
- 查询并发与导出耗时

## 4. 风险与应对
1. 风险：上游 API 波动
- 应对：重试、超时、熔断、失败告警

2. 风险：字段类型复杂导致查询歧义
- 应对：字段备注治理 + raw_json 保底输出

3. 风险：定时任务重复触发
- 应对：任务锁（PostgreSQL advisory lock）

4. 风险：密钥泄漏
- 应对：哈希存储、过期策略、快速停用

## 5. 上线准备清单
1. 生产环境变量配置
2. PostgreSQL 备份策略
3. systemd 服务配置
4. 日志轮转策略
5. 管理员初始账号创建
6. 首批表单配置与试同步

## 6. 首月运行建议
1. 第 1 周：每日检查同步失败日志
2. 第 2 周：优化同步周期与分页参数
3. 第 3 周：完善字段备注，提升 AI 查询效果
4. 第 4 周：形成二期接入 SQLServer/金蝶评估报告
