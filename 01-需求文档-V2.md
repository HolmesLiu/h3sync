# 瑞普数据同步系统 需求文档（V2）

## 1. 项目背景
为郑州瑞普生物工程有限公司建设一个可扩展的数据采集与查询平台。

系统目标：
1. 统一采集多数据源数据（一期：氚云 + MSSQL 备份 SQL 文件）。
2. 提供后台管理（登录、表单配置、同步管理、企业资料库、API Key 管理、日志审计）。
3. 提供 AI 友好的查询 API（带表单备注/说明）。

## 2. 一期范围

### 2.1 氚云采集（保留）
1. 支持按表单配置同步（自动/手动，独立周期）。
2. 支持全量初始化 + 增量更新（不做删除同步）。
3. 后台可查看表单数据（分页、筛选、导出）。

### 2.2 MSSQL 备份采集（新增）
1. 新增菜单：`MSSQL表单管理`。
2. 页面默认展示已配置表单列表，右上角增加 `表单库` 按钮。
3. 表单库配置项：
   - SQL 文件根路径（递归读取子目录）
   - 更新时间（最近扫描时间）
4. 业务规则：
   - 系统扫描根路径下所有 `.sql` 文件。
   - 文件命名中解析：时间、模式（FULL/INCR）、schema.table。
   - 首次遇到某表：自动创建表单配置并初始化导入。
   - 后续按增量文件进行增量更新（仅新增/更新，不删除）。
5. 表单列表支持：
   - 表名绑定
   - 表数据查看
   - 手动同步

### 2.3 查询 API
1. 后台可创建 API Key：名称、备注、权限表单、是否过期、过期时间。
2. API Key 权限粒度：表单级。
3. API 返回包含：表单备注/字段信息，便于 AI 理解语义。
4. 支持标准查询 + 基础关联查询。

### 2.4 企业资料库（新增）
1. 新增菜单：`企业资料库`。
2. 用户可上传 `Excel/CSV/TXT/DOCX/PDF` 文件生成资料表；其中 Excel 按 `Sheet` 拆分：
   - 一个 `Sheet` 生成一个表单。
   - 多个 `Sheet` 一次生成多个表单。
   - 表单名规则：`基础表单名_数字`。
   - 备注规则：同一文件生成的表单共用上传备注。
3. Excel 建表时优先使用首行标题作为字段初始名称；字段代码自动规范化。
4. 企业资料表支持：
   - 查看数据
   - 分页/筛选/导出
   - 新增内容
   - 编辑内容
   - 删除内容
   - 删除表单
   - 编辑表单名称、分组、备注
   - 编辑字段显示名/字段备注，用于标识字段语义
5. 企业资料表支持表内继续导入结构化数据：
   - 入口位于表单内容页
   - 导入前必须校验与当前表单字段格式是否一致
   - 字段数量、字段顺序、字段名称不匹配时禁止导入
6. 上传与解析约束：
   - 同一时刻仅允许一个资料文件处于解析中
   - 同一源文件不可重复上传
   - Excel 多 Sheet 拆表时，共享同一个源文件记录，但每个 Sheet 独立建表
7. 企业资料表属于统一表单体系，可直接绑定到 API Key 权限，并被 OpenAPI 查询接口调用，作为 AI 客服 Agent 的基础知识数据。

### 2.5 日志与审计
1. 同步日志：状态、触发方式、起止时间、条数、错误信息。
2. API 查询日志：密钥、请求摘要、结果条数、耗时、IP。
3. 后台操作日志：操作者、动作、目标、详情、IP。

## 3. 关键业务约束
1. 数据库使用 PostgreSQL。
2. 运行环境为 Ubuntu（当前开发环境可在 Windows）。
3. 表单支持中文备注。
4. 短期内表结构相对稳定，但系统需保留扩展性。
5. 一期先实现氚云 + MSSQL 备份接入，并提供企业资料库文件建表能力；后续预留金蝶等 Connector。

## 4. 数据规模（当前估算）
1. 表单数约 50。
2. 单表约 5000 行。
3. 日增量 10-100 行/表。

## 5. 验收标准（V2）
1. 后台存在两个管理域：`氚云表单管理`、`MSSQL表单管理`。
2. MSSQL 可从配置路径递归发现 SQL 文件并完成首次导入。
3. 对已导入表单可执行增量同步，重复执行不产生脏数据。
4. 企业资料库支持文件建表、字段编辑、内容增删改查、内容导入校验。
5. Excel 多 Sheet 文件可一次拆分生成多个资料表，且字段命名与 Sheet 互不干扰。
6. 表数据可在后台分页查看、筛选、导出。
7. 同步/API/后台操作日志完整可追溯。
