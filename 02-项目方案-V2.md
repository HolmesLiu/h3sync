# 瑞普数据同步系统 项目方案（V2）

## 1. 技术选型（定稿）
1. 语言与框架：Go + Gin（单体架构）。
2. 页面：服务端模板渲染（不做前后端分离）。
3. 数据库：PostgreSQL。
4. 数据访问：sqlx + 手写 SQL。
5. 定时调度：内置 poller（按表单配置周期扫描执行）。
6. 日志：zap 文件日志 + 业务日志入库。
7. 部署：Ubuntu + systemd。

## 2. 架构设计

### 2.1 模块划分
1. `admin`：后台登录、配置管理、数据查看。
2. `sync`：同步调度与执行。
3. `connector/h3`：氚云采集连接器。
4. `connector/mssql_backup`：MSSQL 备份 SQL 文件连接器（新增）。
5. `enterprise_library`：企业资料库文件解析、建表、表内导入（新增）。
6. `openapi`：对外查询 API。
7. `audit/log`：审计与日志。

### 2.2 统一采集抽象
以“表单”为统一中间模型，不论来源是氚云、MSSQL 还是企业资料库：
1. 表单注册信息（名称、备注、分组、同步策略）。
2. 业务数据统一落到 `biz_*` 业务表。
3. 保留 `raw_json` 作为完整原始数据。

### 2.3 MSSQL 备份接入策略
1. 表单库路径配置在系统设置中。
2. 同步器递归发现 `.sql` 文件。
3. 通过文件名识别：
   - 时间戳
   - 模式（FULL/INCR）
   - 来源表（dbo.xxx）
4. 导入策略：
   - 首次：FULL 初始化。
   - 增量：按文件时间顺序导入 INCR。
5. 幂等策略：
   - 记录已处理文件位点。
   - 行级按主键（默认 id）upsert。

### 2.4 企业资料库接入策略
1. 企业资料库使用独立后台入口，但底层仍复用统一 `form_registry + biz_*` 模型。
2. 文件类型支持：
   - `csv/txt/xlsx/xlsm/xltx/docx/pdf`
3. 解析策略：
   - Excel：按 `Sheet` 拆成多个表单，每个 `Sheet` 独立建表、独立字段集
   - CSV：单文件单表
   - TXT/DOCX/PDF：按文本片段拆行，落到结构化内容表
4. Excel 建表规则：
   - 首行标题作为字段初始显示名
   - 字段代码做规范化处理
   - 表单名按“基础表单名 + `_数字`”生成
5. 导入策略：
   - 初次上传：建表并导入内容
   - 表内追加导入：仅允许结构化文件，且必须先校验字段数量、顺序、名称是否与现有表一致
6. 并发与幂等：
   - 同一时刻仅允许一个资料文件解析
   - 同一源文件按校验和去重，避免重复建表
   - 一个工作簿拆成多个表单时，共享源文件元数据，但每个 Sheet 独立记录 `sheet_name`

## 3. 数据设计增量

### 3.1 元数据扩展
1. `form_registry` 增加 `source_type`（H3/MSSQL_BACKUP）。
2. 新增 `system_settings`：保存 `mssql_backup_root_path` 等系统配置。
3. 新增 `mssql_form_registry`：保存源表名、增量字段、最近文件位点、最近扫描时间等。
4. 新增 `enterprise_library_forms`：保存源文件名、存储文件名、sheet_name、源文件校验和、表单校验和、解析状态、行数等。
5. `form_field_registry` 复用到企业资料表字段管理，允许字段显示名/备注二次编辑。

### 3.2 日志复用
`sync_logs` 继续作为统一同步日志，区分来源由表单元数据识别。

## 4. 后台页面方案
1. 左侧菜单：
   - 控制台
   - 氚云表单管理（原表单管理更名）
   - MSSQL表单管理（新增）
   - 企业资料库（新增）
   - API Key 管理
2. MSSQL表单管理页：
   - 默认列表
   - 右上角“表单库”按钮
   - 支持表名绑定 / 查看数据 / 手动同步
3. 企业资料库页：
   - 上传资料并建表
   - 列表按分组展示
   - 显示来源文件、Sheet、解析状态、记录数
   - 支持表单元信息编辑与删除
4. 企业资料详情页：
   - 字段设置区：可修改字段显示名/备注
   - 数据导入区：支持按当前字段结构追加导入
   - 数据区：支持分页查看、筛选、导出、增删改

## 5. 查询 API 方案
1. API Key 权限维持表单级。
2. 查询接口对来源无感，统一查询采集后的业务表。
3. 企业资料库表单与氚云/MSSQL 表单同级对待，可直接授予 API Key 权限。
4. 返回结构中附带表单备注与字段信息，保障 AI 语义可读性。

## 6. 可扩展性设计
1. 以 Connector 方式新增数据源（SQLServer在线库、金蝶）。
2. 同步执行与存储模型复用，减少二期改造成本。
3. 将“文件发现/解析/位点管理”与“数据写入”解耦。
4. 企业资料库保留“解析建表”和“表内增量导入”两条独立链路，便于后续扩展更多文件模板。

## 7. 风险与应对
1. SQL 文件格式不一致：采用容错解析 + 失败文件日志隔离。
2. 大文件导入耗时：分批写入与可中断重试。
3. 重复增量文件：文件位点去重 + 行级 upsert 双保险。
