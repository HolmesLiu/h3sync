# 检测问题

## 待定问题

### 4. 查询 API 与需求不一致
- 需求文档要求“支持标准查询 + 基础关联查询”。
- 当前实现仅支持单表查询，过滤操作只有 `eq` 和 `contains`，多个条件之间是 `AND` 拼接。
- 现状文件：
  - `internal/service/query_service.go`
  - `internal/handler/openapi/handler.go`
  - `01-需求文档-V2.md`

### 5. MSSQL 幂等与导入策略待补强
- 当前已处理文件仅按 `file_path` 去重，`checksum` 没有参与重复判定。
- `FULL` / `INCR` 虽然能从文件名解析，但实际导入流程没有基于模式执行初始化与增量分流。
- 如果同一路径文件被覆盖更新，系统会将其视为“已处理”而跳过，可能导致漏同步。
- 现状文件：
  - `internal/service/mssql_backup_service.go`
  - `internal/repository/form_repo.go`
  - `migrations/0003_mssql_support.sql`
  - `01-需求文档-V2.md`
