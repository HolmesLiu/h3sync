# 瑞普数据同步系统 - 项目方案（V1）

## 1. 技术选型（定稿）
1. 语言与框架：Go 1.23+ + Gin
2. 页面形态：服务端渲染（html/template），不做前后端分离
3. 数据库：PostgreSQL 16
4. 数据访问：sqlx + 手写 SQL
5. 任务调度：robfig/cron v3
6. 数据库迁移：golang-migrate
7. 导出：excelize
8. 日志：zap（文件日志）+ 业务日志入库
9. 部署：systemd 托管单体进程（可选 Nginx 反向代理）

## 2. 总体架构
单体应用按模块拆分：
1. `admin`：后台页面、登录、配置管理
2. `sync`：同步引擎（调度 + 执行 + 游标管理）
3. `connector`：数据源适配器（一期实现 H3Connector）
4. `query-api`：对外查询接口（API Key 鉴权）
5. `audit`：日志与审计

## 3. 模块设计
### 3.1 Connector 抽象
定义统一接口，保证后续可扩展：
- `ListForms()`
- `GetFormSchema(schemaCode)`
- `LoadData(schemaCode, cursor, pageSize)`
- `Ping()`

一期实现：`H3Connector`（基于 EngineCode/EngineSecret）

### 3.2 同步引擎
1. 调度器按 `form_registry` 配置触发任务
2. 手动触发与自动触发共用执行逻辑
3. 增量游标：`last_cursor_modified_time + last_cursor_object_id`
4. 写入策略：按 `object_id` upsert
5. 错误处理：失败记录日志，支持下次继续增量

### 3.3 数据存储
1. 元数据表：表单配置、字段备注、API Key、权限、日志
2. 动态业务表：每表单一张物理表（`biz_{schema_code}`）
3. 数据列：业务字段 + `raw_json` + `modified_time` + `object_id`

### 3.4 后台管理
页面清单：
1. 登录页
2. 首页仪表盘（同步成功率、最近任务）
3. 表单管理页（配置、备注、同步策略）
4. 字段备注管理页
5. 数据查看页（筛选、详情、导出）
6. API Key 管理页（名称、备注、权限、过期、停用）
7. 日志页（同步/API/审计）

### 3.5 查询 API
1. `POST /openapi/query/{schemaCode}`
2. Header：`X-API-Key`
3. 支持过滤、分页、基础关联
4. 响应附带：
- 表单名称 + 表单中文备注
- 字段名称 + 字段中文备注
- 数据结果

## 4. 核心数据表（建议）
1. `admin_users`
2. `form_registry`
3. `form_field_registry`
4. `sync_job_runs`
5. `api_keys`
6. `api_key_form_permissions`
7. `api_query_logs`
8. `admin_audit_logs`
9. `biz_{schema_code}`（动态业务表）

## 5. 安全方案
1. 后台密码使用 bcrypt/argon2 哈希
2. API Key 仅明文展示一次，库内存哈希
3. API Key 支持过期和停用
4. 敏感操作写审计日志
5. SQL 参数化，防注入

## 6. 日志策略
1. 应用日志：zap 输出到文件（按天切分）
2. 同步日志：`sync_job_runs`
3. API 查询日志：`api_query_logs`
4. 后台审计日志：`admin_audit_logs`

## 7. 扩展预留
1. `connector` 插件化，二期新增 `SqlServerBackupConnector`、`KingdeeConnector`
2. 查询 API 保持统一输入输出协议
3. 元数据层不绑定特定数据源，支持多源并存

## 8. 运维与部署建议
1. Ubuntu 创建独立用户运行服务
2. PostgreSQL 独立实例，定期备份
3. systemd 配置开机自启与崩溃重启
4. 使用 `.env` 管理密钥，禁止入库
5. 日志轮转与归档（如 180 天）
